\begin{abstract}
Federated learning servers aggregate client updates without seeing client data, which leaves training exposed to both untargeted model poisoning and targeted backdoor attacks. Most robust aggregators rely on update geometry, coordinate-wise robust statistics, or alignment with a single trusted reference direction. These signals weaken when benign clients are heterogeneous and adaptive attackers stay close to the benign update manifold. We propose \textbf{CARAT}, a class-aware robust aggregation rule built on \emph{hidden task certificates}. Given a small labeled server-side reference set, CARAT samples multiple hidden, class-balanced micro-tasks and scores each client update by the loss reduction it achieves across them. CARAT combines these task-conditioned scores with robust clipping, common-radius evaluation, coordinate-rank anomaly penalties, and capped-simplex weight optimization with temporal regularization. The result is an aggregator that favors updates that remain useful across diverse semantic slices while limiting the influence of structurally extreme clients. We also define an evaluation protocol for CARAT that emphasizes class-rich benchmarks, separate untargeted and backdoor threat models, matched trusted-data budgets, and class-sensitive metrics such as worst-class accuracy and attack success rate.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

Federated learning (FL) lets many clients train a shared model without centralizing raw data \citep{mcmahan2017fedavg,bonawitz2017secureagg,kairouz2021federated}. This decentralization is attractive in privacy- and governance-sensitive settings, but it leaves the server in an asymmetric position: the global model must be updated from client messages that the server cannot directly verify. The aggregation rule therefore becomes a security-critical part of the FL system. Malicious clients can slow convergence, distort the decision boundary, or implant triggers that activate attacker-chosen behavior at test time \citep{bhagoji2019adversarial,fang2020local,shejwalkar2021manipulating,baruch2019alie,bagdasaryan2020backdoor,xie2020dba,wang2020tails,zhang2022neurotoxin}.

The central difficulty is that benign heterogeneity and malicious behavior can look similar from the server's perspective. Under non-i.i.d.\ partitions, benign clients can differ substantially in update direction, norm, sparsity, and class emphasis \citep{zhao2018noniid,li2020fedprox,karimireddy2020scaffold,zhu2021noniidsurvey}. An unusual update may therefore reflect either a harmful client or a legitimately different local distribution. This ambiguity weakens defenses based on coarse notions of centrality. Pairwise-distance methods can mistake benign dispersion for adversarial deviation, coordinate-wise robust statistics discard cross-coordinate structure, and trusted-anchor approaches compress all auxiliary supervision into a single global reference direction \citep{blanchard2017krum,elmhamdi2018bulyan,yin2018median,xie2019zeno,chen2018draco,li2019rsa,pillutla2022rfa,cao2021fltrust}. Such signals are often too coarse to expose attacks whose effect is concentrated on a subset of classes or semantic regions.

CARAT addresses this problem by shifting validation from geometric similarity to task-conditioned utility. The server maintains a small labeled reference set and repeatedly instantiates hidden, class-balanced micro-tasks from it. Each client update is scored by the loss reduction it induces across these tasks, producing a certificate matrix that measures whether the update is broadly useful across semantic slices of the problem instead of merely close to the geometric center of the client population. CARAT then combines this signal with robust clipping, common-radius evaluation, coordinate-rank anomaly penalties, and capped-simplex weight optimization with temporal regularization.

This design targets FL deployments in which a modest server-side audit set is realistic, such as cross-silo or institutionally managed settings. It is not a fully data-free defense. Within that setting, CARAT aims to preserve benign diversity while reducing the influence of updates that are semantically harmful or structurally extreme. Our main contributions are as follows.

\begin{itemize}
\item We propose CARAT, a class-aware robust aggregation rule that validates client updates through hidden-task certificates constructed from server-side labeled reference data, replacing one-shot geometric trust signals with multi-task utility checks.
\item We formulate aggregation as a capped-simplex optimization problem that combines worst-task utility across hidden tasks, coordinate-rank anomaly suppression, and temporal regularization, yielding a robust weighting mechanism compatible with heterogeneous FL.
\item We define an evaluation protocol for CARAT centered on class-rich benchmarks, separate untargeted and backdoor suites, matched trusted-data budgets, and class-sensitive metrics, so that the empirical study directly tests the semantic robustness claims of the method.
\end{itemize}

\section{Related Work}
\label{sec:related-work}

\paragraph{Poisoning attacks.}
Modern FL attacks are designed to remain effective under server-side defenses. Early work already showed that even constrained adversaries can poison the global model while remaining relatively stealthy \citep{bhagoji2019adversarial,fang2020local,shejwalkar2021manipulating}. For untargeted degradation, attacks such as ALIE, MinMax, and MinSum craft malicious updates that evade simple geometric or statistical filters while still harming convergence or accuracy \citep{shejwalkar2021manipulating,baruch2019alie}. For targeted manipulation, model-replacement backdoors, distributed trigger construction, tail attacks, and durable attacks such as Neurotoxin show that an update can remain globally plausible while implanting highly specific malicious behavior \citep{bagdasaryan2020backdoor,xie2020dba,wang2020tails,zhang2022neurotoxin}. Surveys make the broader point explicit: once the server cannot inspect raw client data, poisoning and backdoor attacks become a systems concern rather than an edge case \citep{kairouz2021federated,liu2022threats}.

\paragraph{Robust aggregation under heterogeneity.}
Classical Byzantine-robust aggregation rules infer benignness from centrality. Krum and related geometry-based methods favor updates that remain close to the client population under pairwise distances \citep{blanchard2017krum,elmhamdi2018bulyan}, while coordinate-wise median and trimmed mean suppress marginal extremes independently \citep{yin2018median}. Later variants extend this family through suspicion scoring, regularized aggregation, coding-based redundancy, and geometric-median aggregation \citep{xie2019zeno,li2019rsa,chen2018draco,pillutla2022rfa,yin2019saddle}. These methods are attractive because they require little server-side prior knowledge. Under heterogeneous data, however, they can still confuse legitimate benign dispersion with malicious deviation. This limitation has motivated methods that address heterogeneity more directly through optimization stabilization, resampling, bucketing, local-SGD analysis, gradient splitting, nearest-neighbor mixing, label-skew-aware aggregation, and entropy-based public-data guidance \citep{li2020fedprox,karimireddy2020scaffold,he2021resampling,karimireddy2022bucketing,data2021localsgd,liu2023gas,allouah2023fixing,bao2024boba,huang2024sdea,zhu2021noniidsurvey}.

\paragraph{Trusted-data and backdoor defenses.}
A third line of work augments aggregation with trusted data, history, or representation-level inspection. In collaborative learning, AUROR anticipated update screening for poisoning resistance \citep{shen2016auror}. In FL, FLTrust introduces a trusted server anchor, FoolsGold and FLDetector rely on historical similarity diagnostics, and FLARE, FLAME, DeepSight, CRFL, and FLIP use latent-space inspection, clustering, clipping, smoothing, or reverse-engineering-based backdoor mitigation \citep{cao2021fltrust,fung2020foolsgold,zhang2022fldetector,wang2022flare,nguyen2022flame,rieger2022deepsight,xie2021crfl,zhang2023flip}. Recent work also highlights the cost of robustness itself, including clipping design and the tradeoff between Byzantine resilience and clean accuracy \citep{allouah2025arc,yang2025tension}. CARAT is closest to this broader defense-and-detection family, but it uses a different validation principle: instead of reducing trust to a single anchor direction, geometric similarity, or post-hoc clustering signal, it asks whether a client update remains useful across multiple hidden semantic sub-tasks.

\section{Method}
\label{sec:method}

\subsection{Formulation and Assumptions}

We consider synchronous FL with $n$ clients and a global parameter vector $w_r \in \R^d$ at round $r$. At each round, client $i$ returns a message $u_i^r$, which the server maps to a vector-form update $\Delta_i^r \in \R^d$. For FedAvg, $\Delta_i^r$ is the client model delta relative to the current global model; for gradient-style optimizers, it is the client update itself. The server must aggregate $\{\Delta_i^r\}_{i=1}^n$ into a single global step while a fraction $\rho$ of clients may be malicious and coordinated.

CARAT assumes the server has access to labeled reference data $\mathcal{D}^{\mathrm{ref}}$. From this data it constructs a probe pool $\mathcal{P}$ and repeatedly samples hidden, class-balanced validation tasks. This assumption is stronger than fully data-free robust aggregation and should be treated as part of the threat model, not as an incidental implementation detail. CARAT uses the auxiliary data not to define a single trusted direction, but to generate multiple hidden semantic checks against which candidate client updates can be evaluated.

The objective is to assign aggregation weights that preserve benign diversity while suppressing updates that are semantically harmful or structurally anomalous. This distinction matters in non-i.i.d.\ FL, where benign clients may differ substantially in direction, norm, and class emphasis. Geometric dispersion alone is therefore insufficient. CARAT divides each round into three stages: common-geometry evaluation, task-aware and structural scoring, and robust weight optimization. Evaluation is normalized across clients, while the final server step remains conservative because it is computed from clipped updates.

\subsection{Method Details}

At round $r$, CARAT first converts client messages into vector-form deltas and computes their norms. Before any semantic evaluation, the server robustly clips client updates so that a small number of large-magnitude messages cannot dominate the comparison stage. Let $\|\Delta_i\|_2$ denote the norm of client $i$'s update. In our implementation, the clipping threshold $\tau_r$ is estimated by the MAD-based rule
\begin{equation}
\tau_r = \operatorname{median}(\{\|\Delta_i\|_2\}) +
k \cdot 1.4826 \cdot \operatorname{MAD}(\{\|\Delta_i\|_2\}),
\label{eq:clip}
\end{equation}
and the clipped update is
\begin{equation}
\bar{\Delta}_i = \min\left(1, \frac{\tau_r}{\|\Delta_i\|_2 + \varepsilon}\right)\Delta_i.
\end{equation}

CARAT then rescales each clipped update to a common evaluation radius $q_r$:
\begin{equation}
\tilde{\Delta}_i = q_r \frac{\bar{\Delta}_i}{\|\bar{\Delta}_i\|_2 + \varepsilon},
\label{eq:normalize}
\end{equation}
where $q_r$ is chosen as a quantile of the positive clipped norms. This normalization removes pure scale advantages during hidden-task comparison. The normalized vectors are used only for evaluation; the final model update is still computed from the clipped vectors. CARAT uses normalization to create a fair scoring geometry, not to define the actual optimization step taken by the server.

After preprocessing, CARAT samples $T$ hidden tasks from the probe pool $\mathcal{P}$, with each task constructed to be approximately class-balanced. This is what makes the method class-aware. A malicious update can remain globally plausible while degrading only a subset of classes or semantic regions. Repeated balanced hidden tasks make these localized failures harder to hide. For each hidden task $t$, let $\ell_t(w)$ denote the average cross-entropy loss of model $w$ on that task. CARAT defines a \emph{hidden-task certificate}
\begin{equation}
c_{i,t} = \ell_t(w_r) - \ell_t(w_r + \tilde{\Delta}_i).
\label{eq:cert}
\end{equation}
Here, ``certificate'' denotes an internal task-level utility score, not a formal certified-robustness guarantee. Positive values indicate that client $i$ improves task $t$ relative to the current global model.

Stacking these scores yields a certificate matrix $C \in \R^{n \times T}$. Because different tasks may have different intrinsic difficulty and baseline loss scale, CARAT robustly standardizes each column of $C$ before optimization. This preserves within-task ordering while making scores comparable across hidden slices. The resulting matrix gives the server a richer signal than client-to-client similarity alone: a client should be trusted because it remains broadly useful across hidden tasks, not merely because it lies near the geometric center of the client population.

Task utility alone is insufficient, because a malicious client may overfit the probe tasks or align with part of the semantic signal while remaining structurally extreme in parameter space. CARAT therefore augments hidden-task certificates with a coordinate-rank anomaly score. For a sampled coordinate set $S \subseteq \{1,\dots,d\}$, it ranks clients coordinate-wise and measures the average normalized distance from the center rank:
\begin{equation}
r_i^{(S)} = \frac{1}{|S|}\sum_{j \in S}
\left|
\frac{\operatorname{rank}_{i,j}}{n-1} - \frac{1}{2}
\right|.
\label{eq:rank}
\end{equation}
CARAT repeats this computation over several random coordinate subsets and aggregates the resulting values robustly. A client receives a large anomaly score if its update remains persistently extreme across many coordinates rather than on a few isolated entries. After robust standardization, only positive deviations are retained as penalties. The signal is intentionally asymmetric: suspicious extremeness is penalized, whereas unusual centrality is not explicitly rewarded. Together, the certificate matrix and rank penalty separate two notions that are often conflated in robust aggregation: whether an update is useful and whether it is structurally atypical.

\subsection{Weight Optimization and Aggregation}

CARAT maps these signals to aggregation weights by solving a constrained optimization problem on the capped simplex
\begin{equation}
\mathcal{A}(\hat{\rho}) =
\left\{
\alpha \in \R^n :
\alpha_i \ge 0,\;
\sum_{i=1}^n \alpha_i = 1,\;
\alpha_i \le \frac{1}{(1-\hat{\rho})n}
\right\},
\label{eq:capped-simplex}
\end{equation}
where $\hat{\rho}$ is an estimate of the attacker fraction. The cap prevents any single client from dominating the aggregate and encodes a conservative trust budget at the weight level: even when a client appears useful, it cannot absorb arbitrary influence within one round.

Let $r \in \R^n$ denote the positive rank penalties and let $\alpha^{\mathrm{prev}}$ be the previous round's weights. CARAT solves
\begin{equation}
\max_{\alpha \in \mathcal{A}(\hat{\rho})}
\;\softmin_{\beta}(C^\top \alpha)
- \lambda_r r^\top \alpha
- \lambda_p \|\alpha - \alpha^{\mathrm{prev}}\|_2^2,
\label{eq:objective}
\end{equation}
where
\begin{equation}
\softmin_{\beta}(s_1,\dots,s_T)
= -\frac{1}{\beta}\log\left(\frac{1}{T}\sum_{t=1}^T e^{-\beta s_t}\right).
\label{eq:softmin}
\end{equation}
The first term is a smooth approximation to worst-task utility: large $\beta$ places more emphasis on the weakest hidden tasks, while finite $\beta$ preserves optimization stability. The second term penalizes structurally extreme clients, and the third term regularizes the solution toward the previous round to reduce unnecessary oscillation. Eq.~\eqref{eq:objective} therefore implements a simple decision rule: a client should retain influence only if it remains useful across hidden tasks and does not repeatedly appear as an outlier in rank space.

After optimization, CARAT aggregates the clipped updates:
\begin{equation}
\Delta_{\mathrm{agg}} = \sum_{i=1}^n \alpha_i \bar{\Delta}_i.
\label{eq:aggregate}
\end{equation}
This is the final server step. The aggregate inherits the robustness benefits of clipping while using weights learned from the richer evaluation geometry defined above. In practice, the main additional computation comes from evaluating $n$ candidate updates over $T$ hidden tasks and computing rank penalties on coordinate subsamples. Probe-model forward passes dominate, while the ranking and optimization stages remain tunable through explicit budget parameters.

A compact round-level algorithm summary is provided in Appendix~\ref{sec:appendix-algorithm}. Formal properties of the normalization step, capped-simplex feasibility, softmin behavior, temporal regularization, and per-round complexity are deferred to Appendix~\ref{sec:appendix-theory}.

\section{Experimental Design}
\label{sec:experiments}

We evaluate CARAT on class-rich federated benchmarks to test whether hidden-task certificates improve robustness beyond geometry-only and single-anchor defenses while preserving clean utility under heterogeneous client data. The study is organized around four questions: how much clean-utility cost CARAT incurs without attack, how well it resists untargeted model poisoning as heterogeneity increases, whether the same mechanism reduces class-selective and backdoor failures, and what robustness is obtained per unit of trusted-data budget and runtime overhead. The full benchmark matrix, appendix-only stress tests, and ablation scope are reported in Appendix~\ref{sec:appendix-experiments}.

\subsection{Experimental Setup}

\paragraph{Benchmarks.}
The primary benchmark is CIFAR-100 with ResNet18~\cite{he2016deep}. CIFAR-100 offers a class-rich label space in which hidden semantic slices are meaningful and class-selective failures are easier to expose than on low-cardinality datasets. We complement it with TinyImageNet, again using ResNet18, to test whether the method scales to a larger class vocabulary while keeping the backbone fixed. To anchor the method in a more application-oriented setting, we also include CHMNIST with the same backbone in the appendix, where class-selective degradation has a clearer domain interpretation. CIFAR-10 is retained only as an appendix sanity check rather than a primary source of evidence, because a ten-class benchmark is too coarse to serve as the main validation vehicle for a class-aware defense. Keeping the model family fixed across visual datasets helps attribute observed differences to aggregation rather than to backbone changes.

\paragraph{Threat models.}
For untargeted model poisoning, the main threat models are ALIE~\cite{baruch2019alie}, MinMax~\cite{shejwalkar2021manipulating}, and MinSum~\cite{shejwalkar2021manipulating}. These attacks represent stealthy optimization-based adversaries that try to stay near the benign update manifold while degrading global learning. FangAttack~\cite{fang2020local} is treated as an appendix stress test rather than a core main-text attack because it is especially tied to Krum-style geometry.

The second core suite targets semantic corruption. We focus on ModelReplacement-style backdoors~\cite{bagdasaryan2020backdoor}, DBA~\cite{xie2020dba}, and Neurotoxin~\cite{zhang2022neurotoxin}, because these attacks directly test whether CARAT's hidden tasks can expose updates that preserve apparent global plausibility while damaging specific semantic regions. BadNets-style triggers are retained only as an appendix sanity check, since they are usually easier to detect than adaptive distributed or persistent attacks.

\paragraph{Baselines.}
For untargeted poisoning, the primary baselines are Mean, TrimmedMean~\cite{yin2018median}, MultiKrum~\cite{blanchard2017krum}, FLTrust~\cite{cao2021fltrust}, and CARAT. For targeted and backdoor attacks, the main comparison set is Mean, FLTrust, FLAME~\cite{nguyen2022flame}, DeepSight~\cite{rieger2022deepsight}, and CARAT. This split keeps the baseline families aligned with the dominant defenses in each threat setting rather than forcing one comparison set across all attacks.

\paragraph{Implementation details.}
Across the primary suites, we use $20$ clients with full participation, $300$ global rounds, $4$ local epochs, and learning rate $0.05$. To probe heterogeneity, we evaluate i.i.d.\ partitions together with Dirichlet non-i.i.d.\ regimes $\alpha \in \{0.5,0.1\}$. The main adversarial setting uses an attacker ratio of $20\%$, while $40\%$ is reserved for appendix stress tests. Clean-reference experiments are reported separately from attacked experiments instead of being inserted as a row inside attack tables. Unless otherwise stated, reported means and standard deviations are computed over $3$--$5$ random seeds.

\paragraph{Metrics and fairness.}
Because CARAT is explicitly class-aware, the evaluation must report more than aggregate clean accuracy. For clean utility, we report test accuracy and test loss under no attack. For robustness, we additionally report macro-average accuracy and worst-class accuracy on class-rich benchmarks so that localized failures remain visible. For backdoor experiments, attack success rate (ASR) is always paired with clean accuracy rather than reported in isolation. Fairness requires explicit control of server-side supervision: CARAT and FLTrust are compared under a matched labeled-data budget, and the reference budget is parameterized per class rather than fixed as a dataset-independent total pool size.

\subsection{Main Evaluation Comparisons}

\paragraph{Clean utility.}
We first report no-attack performance to make the cost of robustness visible. These comparisons focus on test accuracy and test loss, with macro and worst-class accuracy added on class-rich benchmarks when aggregate accuracy alone could mask concentrated degradation.

\paragraph{Untargeted poisoning.}
We then compare CARAT against Mean, TrimmedMean, MultiKrum, and FLTrust under ALIE, MinMax, and MinSum across i.i.d.\ and non-i.i.d.\ partitions. The main question is whether task-conditioned validation remains reliable when benign clients are dispersed and adaptive attackers stay close to the benign update manifold.

\paragraph{Targeted and backdoor attacks.}
Finally, we compare CARAT against Mean, FLTrust, FLAME, and DeepSight under ModelReplacement, DBA, and Neurotoxin. These experiments emphasize macro accuracy, worst-class accuracy, and ASR to test whether class-balanced hidden tasks expose updates that remain globally plausible while harming specific semantic regions.

\subsection{Ablations, Stress Tests, and Overhead}

\paragraph{Ablations.}
We ablate the components that correspond directly to CARAT's mechanism: hidden-task count, reference budget per class, rank-penalty weight, temporal-prior weight, clipping controls, and common-radius evaluation. These studies isolate whether the observed robustness comes from class-balanced task validation, structural anomaly suppression, or simply from access to more trusted data.

\paragraph{Stress tests.}
Appendix-only stress tests raise the attacker fraction to $40\%$, add FangAttack and BadNets sanity checks, and vary the client count or class vocabulary. Their role is to probe failure modes and scaling behavior rather than to replace the main-text comparisons.

\paragraph{Overhead analysis.}
Since Proposition~\ref{prop:complexity} identifies probe-model evaluation as the main extra cost, we report runtime per round, total defense overhead, and trusted-data budget sensitivity as first-class evaluation outcomes rather than as implementation footnotes.

\section{Conclusion}
\label{sec:conclusion}

We presented CARAT, a class-aware robust aggregation rule that validates client updates through hidden task certificates rather than relying only on geometric centrality or a single trusted reference direction. By scoring updates across multiple hidden, class-balanced semantic slices and combining those scores with rank-space anomaly suppression and capped-simplex weight optimization, CARAT is designed to preserve benign heterogeneity while reducing the influence of semantically harmful or structurally extreme clients.

The method is most appropriate in FL deployments where the server can maintain a modest labeled audit set and where class-selective failures matter. In that regime, the relevant empirical evaluation centers on class-rich benchmarks, separate untargeted and backdoor threat models, class-sensitive metrics such as macro and worst-class accuracy, and explicit controls for trusted-data budget and runtime overhead. CARAT's central claim is not only that it rejects large outliers, but that it favors updates that remain consistently useful across hidden semantic slices even when attacks are adaptive and benign clients are heterogeneous.

\section{Algorithmic Details}
\label{sec:appendix-algorithm}

\subsection{Round-Level Pseudocode}

Algorithm~\ref{alg:carat-round} summarizes one CARAT server round in standard pseudocode form.

\begin{algorithm}[t]
\caption{CARAT server aggregation at round $r$}
\label{alg:carat-round}
\begin{algorithmic}[1]
\REQUIRE Current global model $w_r$, client messages $\{u_i^r\}_{i=1}^n$, previous-round weights $\alpha^{\mathrm{prev}}$, probe pool $\mathcal{P}$, CARAT hyperparameters
\ENSURE Aggregated update $\Delta_{\mathrm{agg}}$, updated temporal prior $\alpha^{\mathrm{prev}}$
\IF{$\mathcal{P}$ is uninitialized}
    \STATE Build a balanced probe pool from the server-visible training dataset
\ENDIF
\IF{$r=1$}
    \STATE $\alpha^{\mathrm{prev}} \leftarrow \frac{1}{n}\mathbf{1}$
\ENDIF
\FOR{$i=1$ to $n$}
    \STATE Convert $u_i^r$ into a vector-form delta $\Delta_i^r$
\ENDFOR
\STATE Compute the clipping threshold $\tau_r$ from $\{\|\Delta_i^r\|_2\}_{i=1}^n$
\FOR{$i=1$ to $n$}
    \STATE $\bar{\Delta}_i^r \leftarrow \min\!\left(1,\frac{\tau_r}{\|\Delta_i^r\|_2+\varepsilon}\right)\Delta_i^r$
\ENDFOR
\STATE Choose the evaluation radius $q_r$ from a quantile of positive clipped norms
\FOR{$i=1$ to $n$}
    \STATE $\tilde{\Delta}_i^r \leftarrow q_r \bar{\Delta}_i^r / (\|\bar{\Delta}_i^r\|_2+\varepsilon)$
\ENDFOR
\STATE Sample $T$ class-balanced hidden tasks from $\mathcal{P}$ with replacement/subset fallback if strict balanced sampling fails
\FOR{$t=1$ to $T$}
    \STATE Evaluate the baseline loss $\ell_t(w_r)$
\ENDFOR
\FOR{$i=1$ to $n$}
    \FOR{$t=1$ to $T$}
        \STATE $c_{i,t} \leftarrow \ell_t(w_r) - \ell_t(w_r + \tilde{\Delta}_i^r)$
    \ENDFOR
\ENDFOR
\STATE Robustly standardize certificate columns to obtain $C^{\mathrm{std}}$
\STATE Compute rank penalties $r$ from repeated coordinate subsamples of $\{\bar{\Delta}_i^r\}_{i=1}^n$
\STATE $\alpha^{(0)} \leftarrow \alpha^{\mathrm{prev}}$
\FOR{$k=0$ to $K-1$}
    \STATE $z \leftarrow (C^{\mathrm{std}})^\top \alpha^{(k)}$
    \STATE $\pi \leftarrow \softmin$ task weights induced by $z$ and $\beta$
    \STATE $g \leftarrow C^{\mathrm{std}}\pi - \lambda_r r - 2\lambda_p(\alpha^{(k)}-\alpha^{\mathrm{prev}})$
    \STATE $\alpha^{(k+1)} \leftarrow \Pi_{\mathcal{A}(\hat{\rho})}\!\left(\alpha^{(k)}+\eta g\right)$
\ENDFOR
\STATE $\alpha^\star \leftarrow \alpha^{(K)}$ and $\alpha^{\mathrm{prev}} \leftarrow \alpha^\star$
\STATE $\Delta_{\mathrm{agg}} \leftarrow \sum_{i=1}^n \alpha_i^\star \bar{\Delta}_i^r$
\STATE \textbf{return} $\Delta_{\mathrm{agg}}, \alpha^{\mathrm{prev}}$
\end{algorithmic}
\end{algorithm}

\subsection{Implementation Notes}

\paragraph{Inputs.}
Current global model $w_r$, client messages $\{u_i^r\}_{i=1}^n$, previous-round weights $\alpha^{\mathrm{prev}}$, server-visible training dataset, and CARAT hyperparameters.

\paragraph{Persistent state.}
CARAT maintains two pieces of state across rounds: a balanced probe pool sampled at initialization from the server-visible training dataset, and the previous-round weight vector $\alpha^{\mathrm{prev}}$, which serves as the temporal prior for the current round.

\paragraph{Initialization.}
At the first round, the implementation initializes $\alpha^{\mathrm{prev}}$ to the uniform vector. This keeps the initial round unbiased across clients while still allowing later rounds to exploit temporal consistency once CARAT has observed several optimization trajectories.

\paragraph{Reference-pool construction and hidden-task sampling.}
Let $\mathcal{P}_c \subseteq \mathcal{P}$ denote the subset of probe examples with class label $c$. A hidden task is constructed by sampling a small number of examples from each available class pool and concatenating them into one balanced micro-task. If the desired number of samples per class is not available without replacement, CARAT falls back to replacement and, if needed, to a random subset of the probe pool rather than aborting the round. This fail-soft design preserves the intended semantic diversity of the task sampler while ensuring that the server always obtains a well-defined certificate matrix.

\paragraph{Robust score calibration.}
For notational economy, the main text reuses $C$ after standardization. In implementation terms, the raw certificate matrix is first transformed column-wise into
\[
C^{\mathrm{std}}_{i,t}
=
\frac{c_{i,t}-\operatorname{median}_j(c_{j,t})}
{1.4826 \cdot \operatorname{MAD}_j(c_{j,t})+\varepsilon}.
\]
This removes task-specific scale effects while preserving within-task ordering by Proposition~\ref{prop:task-standardization}. Rank penalties are calibrated similarly. For each coordinate subsample $S_s$, let
\[
\rho_i^{(s)}=
\frac{1}{|S_s|}\sum_{j \in S_s}
\left|
\frac{\operatorname{rank}_{i,j}}{n-1} - \frac{1}{2}
\right|.
\]
CARAT then computes a standardized rank score
\[
z_i^{(s)}=
\frac{\rho_i^{(s)}-\operatorname{median}_\ell(\rho_\ell^{(s)})}
{1.4826 \cdot \operatorname{MAD}_\ell(\rho_\ell^{(s)})+\varepsilon},
\]
and aggregates subsamples through
\[
r_i=\max\left(0,\operatorname{median}_s z_i^{(s)}\right).
\]
As a result, a client is penalized only when it is consistently extreme across repeated coordinate views, not because of a single noisy subsample.

\paragraph{Projected optimization details.}
Again writing $C$ for the standardized certificate matrix, CARAT initializes the optimizer with $\alpha^{(0)}=\alpha^{\mathrm{prev}}$ after the first round, and with the uniform vector at initialization. If
\[
z(\alpha)=C^\top \alpha,
\qquad
\pi_t(\alpha)=\frac{\exp(-\beta z_t(\alpha))}{\sum_{\ell=1}^T \exp(-\beta z_\ell(\alpha))},
\]
then the ascent direction used in projected updates is
\[
g(\alpha)=C\pi(\alpha)-\lambda_r r-2\lambda_p(\alpha-\alpha^{\mathrm{prev}}).
\]
With optimizer step size $\eta$, one iteration takes the form
\[
\alpha^{(k+1)}=
\Pi_{\mathcal{A}(\hat{\rho})}
\left(\alpha^{(k)}+\eta g(\alpha^{(k)})\right).
\]
The projection maintains feasibility throughout optimization, so every intermediate iterate already satisfies the nonnegativity, unit-sum, and per-client cap constraints described in the main text.

\paragraph{Round invariants and failure handling.}
Two implementation choices are important for robustness. First, the optimizer never leaves the capped simplex, so every intermediate and final weight vector remains feasible. Second, probe-task construction is fail-soft rather than fail-stop: if strict class-balanced sampling is impossible, the code falls back to replacement or to a random subset, ensuring that the round can still proceed with a well-defined certificate matrix.

\paragraph{Outputs.}
Aggregated server update, updated temporal prior $\alpha^{\mathrm{prev}}$, and diagnostic logs including learned weights, task scores, softmin focus, rank penalties, evaluation radius, clipping threshold, and the identities of the hidden probe tasks used in the current round.

\paragraph{Practical diagnostics.}
The most useful round-level diagnostics for debugging and empirical interpretation are the learned weights, the weakest-task values emphasized by the softmin, the clip threshold $\tau_r$, the evaluation radius $q_r$, and the final rank penalties $r_i$. In successful runs, these quantities help distinguish three qualitatively different cases: benign but diverse clients that retain moderate weight, clearly harmful clients that lose certificate support across multiple tasks, and structurally extreme clients whose influence is reduced primarily by the rank penalty.

\section{Theoretical Analysis}
\label{sec:appendix-theory}

This appendix records the main structural properties used to interpret CARAT. These results are not presented as a complete robustness guarantee against arbitrary adaptive attackers. Instead, they clarify why the method decomposes into a normalized evaluation geometry, a task-aware utility objective, and a constrained robust aggregation step.

\begin{proposition}[Feasibility and support of the capped simplex]
\label{prop:capped-simplex}
Let
\[
\mathcal{A}(\hat{\rho})=
\left\{
\alpha \in \R^n:
\alpha_i \ge 0,\;
\sum_{i=1}^n \alpha_i = 1,\;
\alpha_i \le \frac{1}{(1-\hat{\rho})n}
\right\},
\]
with $\hat{\rho}<1$. Then $\mathcal{A}(\hat{\rho})$ is non-empty, and every $\alpha \in \mathcal{A}(\hat{\rho})$ satisfies
\[
|\operatorname{supp}(\alpha)| \ge (1-\hat{\rho})n.
\]
\end{proposition}

\begin{proof}[Proof sketch]
Let $u = 1/((1-\hat{\rho})n)$. Since $\hat{\rho}<1$, we have $u \ge 1/n$, so the uniform weight vector is feasible and the set is non-empty. For the support bound, if only $k$ coordinates of $\alpha$ are positive, then
\[
1=\sum_{i=1}^n \alpha_i \le k u = \frac{k}{(1-\hat{\rho})n},
\]
which implies $k \ge (1-\hat{\rho})n$. Therefore CARAT can suppress suspicious clients, but cannot collapse the aggregate onto an arbitrarily small subset of participants.
\end{proof}

\begin{proposition}[Softmin bounds and gradient form]
\label{prop:softmin}
Let $s \in \R^T$ and define
\[
\softmin_{\beta}(s)
= -\frac{1}{\beta}\log\left(\frac{1}{T}\sum_{t=1}^T e^{-\beta s_t}\right).
\]
Then
\[
\min_t s_t \le \softmin_{\beta}(s) \le \min_t s_t + \frac{\log T}{\beta}.
\]
Moreover,
\[
\lim_{\beta \to \infty}\softmin_{\beta}(s)=\min_t s_t,
\qquad
\lim_{\beta \to 0}\softmin_{\beta}(s)=\frac{1}{T}\sum_{t=1}^T s_t.
\]
Moreover, if $z(\alpha)=C^\top\alpha$ and
\[
\pi_t(\alpha)=\frac{\exp(-\beta z_t(\alpha))}{\sum_{\ell=1}^T \exp(-\beta z_\ell(\alpha))},
\]
then
\[
\nabla_\alpha \softmin_{\beta}(C^\top\alpha)=C\pi(\alpha).
\]
\end{proposition}

\begin{proof}[Proof sketch]
Let $m=\min_t s_t$. Since
\[
e^{-\beta m}/T \le \frac{1}{T}\sum_{t=1}^T e^{-\beta s_t} \le e^{-\beta m},
\]
applying $-(1/\beta)\log(\cdot)$ yields
\[
m \le \softmin_{\beta}(s) \le m+\frac{\log T}{\beta}.
\]
The limit $\beta \to \infty$ follows immediately from the vanishing gap bound. For $\beta \to 0$, a first-order expansion of the exponential average gives
\[
\frac{1}{T}\sum_{t=1}^T e^{-\beta s_t}
=
1-\beta \frac{1}{T}\sum_{t=1}^T s_t + o(\beta),
\]
so the softmin converges to the arithmetic mean. Differentiating the softmin expression with respect to $z_t$ yields the Gibbs weights $\pi_t(\alpha)$, and the chain rule then gives $C\pi(\alpha)$. Hence the ascent direction is a convex combination of certificate columns, with more weight placed on tasks whose current aggregate utility is smaller.
\end{proof}

\begin{proposition}[Directional invariance of certificate scoring]
\label{prop:directional-invariance}
Suppose two nonzero clipped updates satisfy $\bar{\Delta}_i = a \bar{\Delta}_j$ for some scalar $a>0$. Then their normalized versions in Eq.~\eqref{eq:normalize} coincide, i.e.,
\[
\tilde{\Delta}_i=\tilde{\Delta}_j.
\]
Consequently, their hidden-task certificates are identical:
\[
c_{i,t}=c_{j,t}\qquad \text{for all } t \in \{1,\dots,T\}.
\]
\end{proposition}

\begin{proof}[Proof sketch]
Eq.~\eqref{eq:normalize} rescales every nonzero clipped update to the common radius $q_r$. Positive collinearity is therefore removed by normalization:
\[
q_r\frac{\bar{\Delta}_i}{\|\bar{\Delta}_i\|_2}
=
q_r\frac{a\bar{\Delta}_j}{\|a\bar{\Delta}_j\|_2}
=
q_r\frac{\bar{\Delta}_j}{\|\bar{\Delta}_j\|_2}.
\]
Substituting the resulting equality into Eq.~\eqref{eq:cert} gives identical task scores. Thus, after clipping, the certificate stage depends on direction rather than residual norm magnitude.
\end{proof}

\begin{proposition}[Task-wise standardization preserves within-task ordering]
\label{prop:task-standardization}
Fix a task $t$ and define a standardized certificate score
\[
\hat{c}_{i,t}=\frac{c_{i,t}-m_t}{s_t+\varepsilon},
\]
where $m_t \in \R$ and $s_t>0$ depend only on task $t$ and not on client $i$. Then for any clients $i,j$,
\[
c_{i,t} \ge c_{j,t}
\quad \Longleftrightarrow \quad
\hat{c}_{i,t} \ge \hat{c}_{j,t}.
\]
Moreover,
\[
\hat{c}_{i,t}-\hat{c}_{j,t}
=
\frac{c_{i,t}-c_{j,t}}{s_t+\varepsilon}.
\]
\end{proposition}

\begin{proof}[Proof sketch]
For a fixed task $t$, the map $x \mapsto (x-m_t)/(s_t+\varepsilon)$ is affine with strictly positive slope because $s_t+\varepsilon>0$. Hence it preserves ordering. The difference identity follows by direct subtraction. Therefore column-wise robust standardization calibrates cross-task scale without changing the within-task ranking induced by the raw certificate scores.
\end{proof}

\begin{proposition}[Temporal prior and projected-iterate invariants]
\label{prop:temporal-prior}
Define
\[
f(\alpha)=\softmin_{\beta}(C^\top\alpha)-\lambda_r r^\top\alpha-\lambda_p\|\alpha-\alpha^{\mathrm{prev}}\|_2^2.
\]
If $\alpha,\alpha' \in \mathcal{A}(\hat{\rho})$ satisfy $C^\top\alpha=C^\top\alpha'$ and $r^\top\alpha=r^\top\alpha'$, then
\[
f(\alpha)-f(\alpha')
=
-\lambda_p\!\left(\|\alpha-\alpha^{\mathrm{prev}}\|_2^2-\|\alpha'-\alpha^{\mathrm{prev}}\|_2^2\right).
\]
In addition, under projected gradient ascent
\[
\alpha^{(k+1)}=\Pi_{\mathcal{A}(\hat{\rho})}\!\left(\alpha^{(k)}+\eta \nabla f(\alpha^{(k)})\right),
\]
every iterate remains in $\mathcal{A}(\hat{\rho})$.
\end{proposition}

\begin{proof}[Proof sketch]
The first statement follows by direct substitution: if the certificate and rank terms match, the only remaining difference in objective value is the quadratic prior term. CARAT therefore prefers, among utility-equivalent feasible weights, the solution closer to $\alpha^{\mathrm{prev}}$. The second statement is immediate from the projection operator. Hence the optimizer preserves three round invariants throughout: $\alpha_i \ge 0$, $\sum_i \alpha_i =1$, and $\alpha_i \le 1/((1-\hat{\rho})n)$.
\end{proof}

\begin{proposition}[Norm bound for the final aggregate]
\label{prop:aggregate-bound}
Suppose each clipped update satisfies $\|\bar{\Delta}_i\|_2 \le \tau_r$ and $\alpha \in \mathcal{A}(\hat{\rho})$. Then the final aggregate
\[
\Delta_{\mathrm{agg}}=\sum_{i=1}^n \alpha_i \bar{\Delta}_i
\]
satisfies
\[
\|\Delta_{\mathrm{agg}}\|_2 \le \tau_r.
\]
\end{proposition}

\begin{proof}[Proof sketch]
Because $\alpha_i \ge 0$ and $\sum_i \alpha_i=1$, the aggregate is a convex combination of clipped updates. By the triangle inequality,
\[
\|\Delta_{\mathrm{agg}}\|_2
\le
\sum_{i=1}^n \alpha_i \|\bar{\Delta}_i\|_2
\le
\sum_{i=1}^n \alpha_i \tau_r
=
\tau_r.
\]
Thus the final server step inherits the clipping radius as a hard norm upper bound.
\end{proof}

\begin{proposition}[Per-round complexity]
\label{prop:complexity}
Let $n$ be the number of clients, $T$ the number of hidden tasks, $m$ the number of sampled coordinates for rank scoring, $S$ the number of rank subsamples, and $K$ the number of projected-gradient steps. Ignoring constant factors from batch size and model width, the extra defense-side work of CARAT scales as
\[
\text{probe-model forward passes} + O(Snm + KnT).
\]
\end{proposition}

\begin{proof}[Proof sketch]
CARAT performs one baseline probe evaluation plus $n$ candidate probe evaluations over the $T$ hidden tasks, then computes $S$ rank summaries over $m$ coordinates, and finally executes $K$ projected optimization steps over $n$ weights and $T$ task scores. In the current implementation, the probe-model forward passes dominate wall-clock time, while the rank and optimization stages are lower-order terms controlled by $m$, $S$, and $K$.
\end{proof}

\paragraph{Remark on assumptions.}
The object $\mathcal{D}^{\mathrm{ref}}$ in the main text denotes the labeled server reference set, while the probe pool $\mathcal{P}$ used during each round is a subsampled working set derived from it. In practice, the server first constructs $\mathcal{P}$ from the full server-visible reference data and then samples hidden tasks from $\mathcal{P}$ at each round. This distinction should remain explicit because it is part of the method's server-side assumption and materially strengthens the setting relative to a fully data-free defense.

\section{Additional Experimental Details}
\label{sec:appendix-experiments}

This appendix records the benchmark design, sensitivity analyses, and reporting scope that support the main-text experimental design.

\subsection{Experimental Goals}

The empirical study is designed to test properties specific to CARAT rather than to benchmark it as a generic outlier filter. In particular, we ask whether hidden-task validation improves robustness when benign clients are heterogeneous, whether that advantage persists on class-rich benchmarks where class-selective failure matters, and how much robustness is purchased per unit of trusted-data budget and runtime overhead.

\subsection{Benchmarks}

\paragraph{Primary class-rich benchmark.}
The main benchmark is CIFAR-100 with ResNet18. This setting best matches CARAT's motivation because the label space is rich enough for different hidden tasks to emphasize different semantic slices, and because average clean accuracy alone can hide substantial per-class degradation.

\paragraph{Scalability benchmark.}
TinyImageNet with ResNet18 is used to test whether the same aggregation rule remains effective as the number of classes and the visual vocabulary increase. This benchmark is particularly useful for checking whether the hidden-task construction and reference-budget design remain meaningful beyond CIFAR-100.

\paragraph{Application-oriented benchmark.}
We include CHMNIST with ResNet18 as an application-style benchmark in the appendix. Its role is not to maximize scale, but to show that class-selective failures have a practical interpretation in a domain where server-side auditing with a small labeled reference set is plausible.

\paragraph{Sanity-check benchmark.}
CIFAR-10 is retained only as a sanity-check benchmark. It remains useful for verifying that CARAT does not fail on standard low-cardinality image classification, but it is not treated as a primary source of evidence because its ten classes are too few to fully exercise the method's class-aware design.

\subsection{Threat Models}

\paragraph{Untargeted poisoning.}
The main untargeted suite contains ALIE, MinMax, and MinSum. Together they represent adaptive model-poisoning attacks that aim to remain stealthy under server-side defenses. FangAttack is evaluated only in the appendix as a stress test, since it is especially informative for geometry-based defenses and should not dominate the design of CARAT's main evaluation.

\paragraph{Targeted and backdoor attacks.}
The main targeted suite contains ModelReplacement~\citep{bagdasaryan2020backdoor}, DBA~\citep{xie2020dba}, and Neurotoxin~\citep{zhang2022neurotoxin}. These attacks are better aligned with CARAT's intended advantage because they can preserve global plausibility while damaging specific semantic regions or maintaining durable triggers. BadNets can be reported as an auxiliary sanity-check attack in the appendix, but it is not the main stress case.

\subsection{Baselines}

\paragraph{Untargeted defenses.}
For untargeted poisoning, the primary baselines are Mean, TrimmedMean, MultiKrum, FLTrust, and CARAT. This set spans plain averaging, coordinate-wise robust statistics, geometry-based filtering, and trusted-data anchoring. Methods such as FLDetector or TriGuardFL can be added in the appendix when the goal is to compare against historical-trajectory or class-wise screening ideas, but they are not required in the core main-text comparison.

\paragraph{Backdoor defenses.}
For targeted and backdoor attacks, the primary comparison set is Mean, FLTrust, FLAME, DeepSight, and CARAT. These baselines better reflect the representation-level and clustering-based defenses commonly used in backdoor settings than purely Byzantine-oriented rules do.

\subsection{Metrics and Fairness}

No-attack clean utility is reported separately from attacked runs. The clean metrics are test accuracy and test loss. On class-rich datasets we additionally report macro-average accuracy and worst-class accuracy, because the motivation for CARAT is precisely that averaging can hide semantically concentrated failures. For backdoor experiments, ASR is always reported together with clean accuracy. Runtime per round and total defense overhead are reported for all defenses that add nontrivial server-side computation.

Trusted-data fairness is controlled explicitly. CARAT and FLTrust are compared under the same labeled server-reference budget, rather than allowing one method to benefit from more public data. In addition to fixed-budget comparisons, we sweep the reference budget in the appendix to show whether CARAT's advantage comes from the hidden-task validation principle rather than from budget alone.

\subsection{Shared Training Protocol}

Unless stated otherwise, the shared protocol uses $20$ clients with full participation, $300$ communication rounds, $4$ local epochs, and learning rate $0.05$. The primary heterogeneity settings are i.i.d.\ data and Dirichlet non-i.i.d.\ partitions with $\alpha \in \{0.5, 0.1\}$. The main attacker ratio is $20\%$, while $40\%$ is reserved for appendix stress tests. The primary experiments use $3$--$5$ random seeds per configuration. Scaling studies in the appendix may additionally vary the number of clients or the attacker ratio, but these are treated as stress tests rather than as the core evaluation axis.

\subsection{CARAT Hyperparameters, Stress Tests, and Ablation Plan}

The current implementation defaults are $8$ hidden tasks, $4$ probe samples per class, probe batch size $128$, probe-radius quantile $0.5$, MAD clipping with $k=2.5$, default attacker prior $\hat{\rho}=0.2$, certificate temperature $\beta=12.0$, rank weight $0.20$, prior weight $0.05$, $4096$ sampled coordinates, $5$ rank subsamples, $60$ projected optimization steps, and optimizer learning rate $0.5$.

Two design principles matter when transferring these defaults across datasets. First, the reference budget should scale with the number of classes rather than be fixed as a single dataset-independent total pool size. Second, CARAT should be ablated along the axes that correspond directly to its mechanism: hidden-task count, reference budget per class, rank-penalty weight, temporal-prior weight, and clipping controls. The appendix therefore includes ablations that isolate whether the observed robustness comes from class-balanced hidden tasks, from the structural rank penalty, or simply from access to more trusted data. It also records high-attacker stress tests, scaling analyses, and runtime breakdowns so that robustness, efficiency, and trusted-data dependence are evaluated together.
